# Keel3 Brand Standards (for the shareable methodology document)

These are the brand standards for the client-facing methodology one-pager. Note that this document uses Arial, not Inter; the weekly brief itself uses Inter, but the shareable methodology doc uses Arial for reliable rendering on recipients' machines.

## Colors

- **Navy `#0F1E2E`** — primary headers, title block, stage section headings
- **Orange `#F25C2A`** — accent rules, stage numbers, emphasis
- **Cream `#F0EFE9`** — callout or sidebar background, if used
- **Slate `#3A4A5C`** — secondary and supporting text
- White or very light background for the body

## Typography

- **Arial throughout.** Headers bold, body regular.
- Title large and bold in Navy.
- Stage numbers can be large and Orange as a visual device.
- Comfortable line spacing; this is a document meant to be read, not a dense brief.

## Layout

- One to two pages.
- Title block at top with the brief name "Heads Down to Heads Up" and a subtitle like "How the Brief Is Built."
- A short overview paragraph.
- Five numbered stages, each with a short heading and one paragraph.
- A closing statement on editorial standards.
- A discreet Keel3 footer ("Prepared by Keel3, LLC" or similar).

## Editorial rules

- **No em dashes anywhere.** Use colons, commas, or reword. Standing Keel3 rule.
- Plain language. No hype vocabulary (disrupt, synergy, seamless, game-changer, AI-powered, next-gen, unlock).
- Confident but not boastful. The document should convey rigor through specifics about the process, not through adjectives.

## What stays out of the client-facing version

Never include in a shareable document: specific client names and their sensitivities, internal tool and infrastructure names, internal failure-mode history, or the internal shared/bespoke production economics. See SKILL.md "What to never include in a shareable version" for the full list.
